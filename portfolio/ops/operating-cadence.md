# CIO Operating Cadence

## Daily loop
### Goal
Detect material state changes without becoming hostage to noise.

### Inputs
- live panel
- trigger matrix
- relevant market/macro/news checks

### Output
- update panel only if state changed
- log alert only if a trigger escalated to `review` or `decide`

## Weekly loop
### Goal
Re-synthesize the portfolio by factor, ownership, and trigger pressure.

### Output
- short CIO note
- changes in monitoring priority
- list of positions that need deeper work

## Monthly loop
### Goal
Reassess the analytical agenda and portfolio operating posture.

### Output
- refresh top risks
- refresh deep-dive order
- review whether current cash and concentration are intentional
- review special structures and wrappers

## Quarterly / earnings-season loop
### Goal
Handle the highest-density information periods with discipline.

### Output
- prioritize P1 positions first
- update thesis status where evidence changed
- route only material changes to the CIO decision layer

## Event-driven override
### Goal
Break routine when facts change enough.

### Trigger examples
- earnings surprise
- guidance cut
- management change
- regulatory action
- macro shock
- concentration jump
- kill switch event

## Core rule
Routine exists to reduce drift; event-driven overrides exist to prevent routine from becoming blindness.
