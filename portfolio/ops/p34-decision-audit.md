# P34 Audit — Decision Traceability

## Current problem
The system can already describe positions, triggers, and cadence, but it still lacks a durable answer to:
- what exactly did the CIO decide?
- why?
- based on which evidence?
- what invalidates that decision?
- when must it be revisited?

## Risk if left unresolved
Without a decision log, the committee becomes articulate but forgetful.
That produces:
- repeated debates
- hidden reversals
- sizing drift
- inability to distinguish conviction from improvisation

## Required components
- decision schema
- action-state workflow
- living decision ledger
- template for new decisions
- integration back into live panel and review cadence
