# Portfolio Ops

Operational layer for active portfolio management.

## Files
- `live-position-panel.csv` — daily management cockpit
- `live-position-panel.md` — narrative overview of the cockpit
- `live-panel-schema.md` — canonical panel schema
- `status-vocabulary.md` — stable operational states
- `trigger-taxonomy.md` — trigger classes and escalation rules
- `trigger-matrix.csv` — explicit trigger matrix by position and factor
- `monitoring-triggers.md` — trigger summary for the CIO
- `operating-cadence.md` — daily/weekly/monthly/event-driven cadence
- `daily-monitor-playbook.md` — daily loop checklist
- `weekly-cio-note-template.md` — weekly synthesis output
- `monthly-review-template.md` — monthly operating review output
- `event-driven-review-playbook.md` — override playbook when facts change
- `cio-decision-schema.md` — canonical decision-record structure
- `cio-decision-workflow.md` — how triggers become explicit decisions
- `cio-decision-log.csv` — current decision ledger
- `cio-decision-template.md` — template for new CIO decisions
- `p31-live-panel-audit.md` — audit notes from panel creation
- `p32-trigger-audit.md` — audit notes from trigger design
- `p33-cadence-audit.md` — audit notes from cadence design
- `p34-decision-audit.md` — audit notes from decision-traceability design

## Rule
If the question is **"what do we hold and how are we treating it right now?"**, start here.
