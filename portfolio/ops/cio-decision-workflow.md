# CIO Decision Workflow

## Sequence
1. trigger or review creates pressure
2. owner reframes the issue
3. CIO determines whether this is monitor / review / decide
4. if `decide`, a decision-log entry is created
5. live panel is updated to reflect the active action state
6. next review condition is recorded

## Decision classes
- position sizing decision
- portfolio funding decision
- risk reduction decision
- thesis defense decision
- thesis downgrade decision
- cash / hedge posture decision

## Core rule
The committee should not change posture silently.
If the posture changed, it must be visible in the decision log.
